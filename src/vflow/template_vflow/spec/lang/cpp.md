# C++ 规范附录

> 适用：所有 C++ 代码（基于 C++11/14）。通用原则见 common/，此处只收 C++ 特有条目
> 级别：【规】违反报 CRITICAL/WARNING；【建】违反报 SUGGESTION

## 1. 文件与头文件

1. 【规】头文件扩展名 `.h`；C++ 实现文件 `.cpp`，C 实现文件 `.c`。(源:规则2/3)
2. 【规】源文件统一 UTF-8 无 BOM 格式。(源:规则4)
3. 【规】类的头文件以类名命名且全小写，对应实现文件同名。(源:规则5/6)
4. 【规】用 `#pragma once` 防止头文件重复包含。(源:规则18)
5. 【建】include 顺序：相关头文件 → 项目内头文件 → 其他库头文件 → 标准库头文件，类别间空行分隔。(源:建议17)
6. 【规】用户头文件用 `#include "filename.h"`；系统/编译器头文件用 `#include <filename.h>`。(源:规则20)
7. 【建】include 路径不使用相对路径（`.`/`..`），从项目 include 根写起。(源:建议16/规则21)
8. 【规】仅通过指针或引用访问的类用前置声明，不 include 其头文件（减少编译依赖）。(源:规则19)
9. 【规】不在头文件里定义变量（多重定义风险）。(源:规则63)
10. 【规】文件最后一行必须是空行。(源:规则8)

## 2. 命名约定

11. 【规】命名空间、类、结构体、联合体、枚举类型、模板及模板参数：大写开头驼峰 `ClassNameExample`。(源:规则22/23/28)
12. 【规】函数名大写开头驼峰 `FunctionName()`；形参小写开头驼峰 `formalParameter`。(源:5.3节)
13. 【规】局部变量小写开头驼峰；禁单字符名（i/j/k 作循环变量除外）；禁下划线作分隔。(源:规则25)
14. 【规】成员变量小写开头驼峰 + 尾下划线：`name_`。(源:规则26)
15. 【规】常量小写开头驼峰，带 const 前缀修饰：`const int maxRetryCount`。(源:规则27)
16. 【规】宏名全大写下划线分隔：`MACRO_NAME_EXAMPLE`。(源:规则29)
17. 【规】枚举值全大写下划线分隔（unscoped 带类型前缀如 `COLOR_RED`；scoped 可直接 `RED`）。(源:规则24)
18. 【规】全局名称禁止以 `_` 或 `__` 开头（编译器预留）。(源:规则34)
19. 【建】typedef/using 别名命名同变量，以 `_t` 结尾。(源:建议21)
20. 【规】访问器命名：只读访问器与变量同名 `value()`；读写成对用 `getValue()`/`setValue()`。(源:规则54)

## 3. 现代 C++（C++11/14）

21. 【规】用 `nullptr`，不用 NULL 或 0（避免重载解析和模板推导问题）。(源:规则67)
22. 【建】优先 scoped enum（`enum class`）：强类型、不污染命名空间、可前置声明。(源:建议67)
23. 【建】用 `using` 声明类型别名代替 typedef（更直观、支持模板别名）。(源:建议20/规则69)
24. 【建】局部变量可用 `auto` 绕过繁琐类型名（仅限局部；可读性差时写明类型）。(源:建议37)
25. 【建】优先统一初始化（花括号）：防类型收窄、免疫 most vexing parse；注意 initializer_list 重载陷阱。(源:建议30/38)
26. 【建】编译期常量尽量用 `constexpr`。(源:建议36)
27. 【规】所有虚函数加 `virtual`；所有重写函数加 `override`；禁止再重写的加 `final`。(源:规则110/111/112)
28. 【规】禁用某函数用 `=delete`（声明为 public + delete，错误信息更准确）；可过滤无意义的隐式转换重载。(源:规则109/建议62)
29. 【建】需要拷贝/移动语义时用 `=default` 显式声明（声明析构会抑制移动操作的自动生成——隐性性能陷阱）。(源:建议73)
30. 【规】Lambda 禁用默认捕获，所有捕获显式写出（默认引用捕获易悬空，默认值捕获造成自包含假象）。(源:规则98)
31. 【建】用 `auto` 持有闭包而非 std::function（更小更快）。(源:建议66)
32. 【建】不修改内容的迭代器用 `const_iterator`（cbegin/cend）。(源:建议87)
33. 【建】容器插入考虑 emplace 系列替代 insert/push_back（省临时对象）。(源:建议93)

## 4. 变量与常量

34. 【规】变量声明在尽可能小的作用域，且声明时必须初始化。(源:规则58/62)
35. 【规】一条语句只声明一个变量（指针/引用修饰符易遗漏）。(源:规则37/61)
36. 【规】`*`/`&` 修饰符紧跟类型名：`int* ptr`（配合一行一变量无歧义）。(源:规则38)
37. 【规】C++ 代码禁止全局变量；全局常量/变量封装进类或命名空间。（嵌入式 C 例外见 modules/embedded.md）(源:规则59/33)
38. 【规】用 const/constexpr/enum 定义常量，不用 `#define`。(源:规则56)
39. 【规】指针使用前必须初始化；释放后立即置 `nullptr`；禁止返回指向非动态分配局部变量的指针。(源:规则39/40/41/145)

## 5. 函数设计

40. 【规】非基本类型参数用 const 引用传递（`const Foo&`），不按值传；函数体内不修改的指针参数加 const。(源:规则86/95)
41. 【规】禁止结构体按值传参。(源:规则89)
42. 【建】参数优先引用而非指针；仅当参数可为空时才用指针。(源:规则85)
43. 【规】总是显式指定函数返回类型；决不返回局部变量的引用或指针。(源:规则92/93)
44. 【规】缺省参数只允许用于非虚函数，且值必须始终一致（虚函数缺省参数按静态类型解析——陷阱）。(源:规则88)
45. 【建】避免 bool 型参数（调用处语义不明，不利扩展）——用枚举替代。(源:建议61)
46. 【规】数组作参数时不指定大小，配长度参数传递。(源:规则87)
47. 【规】重载函数的所有版本必须语义一致（同名同目的）。(源:规则90)
48. 【规】形参名在声明和定义中必须一致且不可省略（接口即文档）。(源:规则91)
49. 【建】参数顺序：输入参数在前，输出参数在后。(源:建议59)
50. 【建】避免可变参数（省略号）。(源:建议58)
51. 【建】文件内私有函数放入匿名命名空间（C++）或声明 static（C）。(源:建议56/规则82)

## 6. 类设计

52. 【建】成员按 public → protected → private 顺序定义；各段内按类型→常量→构造→析构→函数→数据排列。(源:建议34/35)
53. 【规】类定义中不写成员函数的实现（模板除外）——内联与否在类外控制。(源:规则53)
54. 【规】不暴露 public/protected 成员数据；派生类需要时提供 protected 访问函数。(源:建议72/建议79)
55. 【规】公共成员函数决不返回成员数据的非 const 引用或指针。(源:规则108)
56. 【规】含指针成员的类必须显式处理拷贝构造和赋值（实现或 =delete）。(源:规则114)
57. 【规】作为基类的类、或含虚函数的类，必须定义虚析构函数。(源:规则115)
58. 【规】析构函数决不抛出异常（内部 try-catch 消化）。(源:规则116/162)
59. 【规】构造函数和析构函数中不调用虚函数。(源:规则117)
60. 【规】单参数非拷贝构造函数必须加 `explicit`，防隐式转换。(源:规则118)
61. 【规】不修改对象状态的成员函数声明为 const；缓存/锁等需修改的成员用 mutable。(源:规则113)
62. 【规】所有继承必须是 public 的；想私有继承时改用组合。(源:规则126)
63. 【建】只在"is-a"关系用继承，"has-a"用组合；不过度使用实现继承。(源:规则127)
64. 【规】多重继承仅允许"除第一个外其余都是纯接口类"。(源:规则129)
65. 【建】内联函数 ≤10 行；构造/析构函数不内联（隐含调用链比表面长）。(源:建议74/规则121)
66. 【规】赋值运算符返回被赋值对象的非常量引用；实现尽量用 copy-and-swap 保证异常安全。(源:规则122/123)
67. 【规】不重载 `&&`、`||`、`,` 运算符（破坏短路求值和求值顺序）。(源:规则124)
68. 【建】重载运算符语义自然统一；相反运算符（==/!=）成对定义。(源:建议76/77)

## 7. 内存与资源

69. 【规】尽可能使用 RAII 管理资源。(源:规则146)
70. 【规】独占所有权用 `std::unique_ptr` + `std::make_unique`；共享所有权用 `std::shared_ptr` + `std::make_shared`。(源:规则147/148/149)
71. 【规】禁止用同一原始指针构造多个 shared_ptr（双控制块→双重释放）；类内部需要用 enable_shared_from_this。(源:规则150)
72. 【规】禁用 std::auto_ptr（已废弃）。(源:规则152)
73. 【规】C++ 代码禁用 malloc/realloc/free；new/delete 与 malloc/free 决不混用。(源:规则141)
74. 【规】数组释放必须用 `delete[]`。(源:规则142)
75. 【规】不重定义全局 new/delete；类重载 new 时必须同时重载 delete。(源:规则140/建议92)
76. 【规】不分配内存并期望调用者释放（接口复杂且易泄漏）——用智能指针或 RAII 容器返回。(源:规则144)

## 8. 异常处理

77. 【规】用 C++ 异常机制报告程序异常状态，代替错误码返回值（库边界/C 接口除外，见 modules/bindings.md）。(源:规则159)
78. 【规】禁用异常的场景：异步事件（信号/中断）、简单可就地处理的错误、流程控制（代替 switch）、不可恢复错误（交给系统）。(源:规则160)
79. 【规】异常类总是派生自 std::exception 或其子类；优先用标准异常类。(源:规则161)
80. 【规】按值抛出，按引用捕获（防拷贝和切片）。(源:规则163)
81. 【规】不使用 C++98 异常规范 throw(...)；确定不抛异常的函数声明 `noexcept`（尤其 move 和 swap）。(源:规则164/165)
82. 【建】编译期可发现的契约用 `static_assert`。(源:建议95)

## 9. 流程与表达式

83. 【规】switch 必须有 default 分支（放最后）；case 必须以 break/return/throw 终止（多标签共享处理除外）。(源:规则73/74)
84. 【规】绝不使用 goto。(源:规则75)
85. 【规】for 循环体内不得修改循环控制变量。(源:规则76)
86. 【建】if/while/switch 条件中不使用 ++/--。(源:建议46)
87. 【规】用括号明确运算符优先级，不依赖默认优先级：`(high << 8) | low`。(源:规则49/71)
88. 【建】区间表达用下闭上开：`x >= 23 && x < 43`。(源:建议45)
89. 【规】sizeof 内不写表达式（不会执行）；优先 `sizeof(varname)` 而非 `sizeof(type)`。(源:规则79/建议48)
90. 【建】独立使用时优先前缀 ++/--（后缀产生临时对象）。(源:建议43)
91. 【建】非 bool 类型不依赖隐式转 bool 判断：`if (count != 0)` 优于 `if (count)`（bool 类型和指针判空除外）。(源:规则72，裁决降为建议)
92. 【建】流程控制语句（if/else/while/for）总是跟 {} 块，即使单语句；空块注释 `// do nothing`。(源:建议33)

## 10. 类型转换与 RTTI

93. 【规】用 C++ 风格转换（static_cast/dynamic_cast/const_cast/reinterpret_cast），不用 C 风格强转。(源:规则138)
94. 【建】避免 reinterpret_cast（底层硬件/系统接口例外）；避免 const_cast。(源:建议90/规则139)
95. 【建】慎用 RTTI：根据类型分支的需求优先用虚函数或访问者模式解决；基于类型的判断树是设计偏轨的强信号。(源:建议91)

## 11. 宏

96. 【规】宏定义表达式用完备括号：`#define AREA(a, b) ((a) * (b))`。(源:规则136)
97. 【规】多语句宏包在 `do { ... } while (0)` 中。(源:规则137)
98. 【建】宏定义 ≤10 行；不在 #include 里使用宏；优先内联函数代替函数式宏。(源:建议88/规则135/建议64)
99. 【建】注意宏参数副作用：`SQUARE(a++)` 会展开成两次自增。(源:建议89)

## 12. 命名空间

100. 【规】类库的所有全局可见标识符必须位于该库唯一的命名空间中。(源:规则153/22)
101. 【规】头文件中禁止 `using namespace`（包括 std）；#include 之前禁止 using。(源:规则154/155)
102. 【规】禁用 inline namespace；禁止在 std 命名空间内声明任何东西。(源:规则157/158)
103. 【建】一个源文件尽量只定义一个命名空间；命名空间右括号后注释名字。(源:建议94/规则156)

## 13. 多线程与并发（C++11）

104. 【建】优先基于任务（std::async）而非基于线程（std::thread）编程；需要线程优先级/线程池等底层控制时才用 thread。(源:建议97)
105. 【规】保证 std::thread 对象在任何路径被析构时都处于 unjoinable 状态（joinable 析构 = 程序终止）。(源:规则166)
106. 【规】std::thread 成员变量声明在类的最后（构造即运行，需保证其他成员已初始化）。(源:规则167)
107. 【建】const 成员函数应当线程安全（多线程可能并发调用同一对象的 const 函数）。(源:建议98)
108. 【建】一次性事件通信考虑 std::promise/future（替代条件变量+flag）。(源:建议99)

## 14. 模板

109. 【建】用 `typename` 声明模板类型参数（class 易误导）。(源:建议80)
110. 【建】函数模板优先按值传参，除非：参数不可拷贝、用于返回数据、需要转发、有显著性能收益。(源:建议81)
111. 【规】类模板对模板参数的成员函数要求必须写入文档。(源:规则132)
112. 【规】成员函数模板不得比预定义的拷贝/移动构造或赋值更优匹配（劫持陷阱）。(源:规则131)

## 15. 可移植性

113. 【规】决不依赖实现定义的、未指定的或未定义的行为（求值顺序、int 大小、char 符号性、除零、野指针）。(源:规则168)
114. 【规】公共接口和可移植代码中禁用平台特定类型（DWORD/TCHAR/LPTCSTR）。(源:规则169)
115. 【规】需要固定大小的类型用 <cstdint>（int32_t 等）；不假设 int 和 long 同大小、指针可放进 long。(源:规则170)
116. 【规】考虑字节序和对齐差异；平台特定代码与可移植代码明确分离。(源:规则171/172)
117. 【建】产品代码保证至少能在另一编译器和平台上编译（MSVC ↔ GCC），无跨平台需求的插件例外。(源:建议2)

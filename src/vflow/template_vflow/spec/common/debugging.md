# 调试方法论

> 适用：所有语言。约束 AI 与人的调试行为（源:vflow新增，参考 helloagents 调试规范）

1. 【规】禁止未经根因分析直接修复。先定位、再分析、后修复；"试一下这个改动行不行"不是方法。
2. 【规】调试时提出 3-5 个可证伪假设，每个假设写清："如果原因是 X，那么改变/观测 Y 会出现 Z"。
3. 【规】单变量验证：一次只改一个变量，否则无法归因。
4. 【规】失败升级阶梯：同方向连续失败 3 次 → 停下，质疑当前思路甚至架构层面原因；失败 4 次 → 扩大排查范围（重读 git 历史、相邻模块）；失败 5 次 → 停止自主尝试，向用户呈现已排除项和 2-4 个候选方向。
5. 【规】修复完成后清理：临时打印、观测点、实验性代码必须删除。
6. 【建】修复有价值的 bug 后做根因沉淀：该类 bug 能否通过规范/检查防止复发？值得的话用 vflow-spec 回写规范库。

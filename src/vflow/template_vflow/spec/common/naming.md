# 命名原则

> 适用：所有语言。各语言的具体大小写约定见 lang/ 对应附录

1. 【建】名字应自描述且尽可能短；超过 30 字符仍不能表意的，考虑用注释补充。(源:建议23)
2. 【建】名字应该是可以读出来的——不能读的名字难以用于讨论。(源:建议24)
3. 【建】互斥意义的变量/相反动作的函数用正确的反义词组命名：add/remove、begin/end、create/destroy、open/close、min/max、send/receive、lock/unlock、show/hide 等。(源:建议25)
4. 【建】保持一致：同一个概念在所有地方用同一个名字（用 name 就处处是 name，不要一处又叫 id）。(源:建议27)
5. 【建】函数名与功能精确贴合，不实现与名字不符的功能。(源:建议28)
6. 【规】标识符中不使用无含义的数字区分（`value1`/`value2` 禁止；`Point3D` 中数字有含义则允许）。(源:规则30)
7. 【规】名字中不使用非普遍接受的缩写词；使用特殊约定或缩写须在文件开头注释说明。普遍接受缩写参照：impl/param/ptr/val/msg/tmp/len/cnt/ret/recv 等。(源:规则35/13+附录C)
8. 【建】避免在名称中做类型编码（禁匈牙利命名如 lpcszName）。现代 IDE 和类型推导已使其失去意义。(源:建议26)

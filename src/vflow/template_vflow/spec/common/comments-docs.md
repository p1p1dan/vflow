# 注释与文档

> 适用：所有语言

1. 【规】所有代码注释使用中文描述（引入的与代码无关的第三方注释可保持原样）。(源:规则11)
2. 【规】注释定性要求：每个类、公共/保护方法、公共枚举/结构体必须有注释；复杂逻辑（算法核心、非直观分支）必须有注释；自解释代码不强制。放弃注释量比例指标。(源:建议7+规则12，裁决改定性)
3. 【建】战略注释（段落用途，放代码前）优先于战术注释（行尾说明）；战术注释过多会让代码无法阅读。(源:4.6章)
4. 【规】每个源码文件必须有文件头注释：版权、文件名、作者、简述（Doxygen 格式 @copyright/@file/@author/@brief）。(源:规则9/10)
5. 【规】公共接口注释使用 Doxygen 标签（@brief/@param/@return/@note），使源码可自动生成参考手册。(源:规则12，原TODO已补全)
6. 【建】边写代码边注释，改代码同步改注释；失效注释必须删除。(源:建议8)
7. 【规】注释放在被描述代码的上方（空行隔开）或右方，不放下方；不在一行代码中间插注释。(源:规则17/15)
8. 【规】注释掉的代码段不得签入：确实无效的删除；必须保留的用 `#if 0` 并写明原因。(源:4.6章)
9. 【建】有物理含义的变量/常量，命名不充分自注释时必须注释其物理含义（工业视觉常见：像素/毫米、角度单位、坐标系）。(源:建议10)
10. 【规】注释符与注释内容之间加空格。(源:规则7)

# 结构与复杂度阈值

> 适用：所有语言。量化阈值供 vflow-review 机械检查

1. 【建】函数长度 ≤50 行为理想（一屏可读完）；硬上限 200 行，超过必须拆分。(源:建议50/51)
2. 【建】一个函数只做一件事（单一职责）。(源:建议52)
3. 【建】控制结构嵌套 ≤3 层。(源:建议50)
4. 【规】圈复杂度 ≤15（判定数+1：if/for/while/do/switch-case/catch/三元/&&/||）。超限必须拆分或重构。(源:规则80，原文"待确定"，裁决取15)
5. 【规】函数参数 ≤5 个；过长参数列表提示设计不良，考虑聚合为结构体。(源:规则84)
6. 【建】多段代码重复做同一件事时，提取为函数。(源:建议53)
7. 【建】减少函数自身或函数间的递归调用（栈空间、可测性、死循环风险）；递归函数名以 `_r` 结尾。(源:建议54/22)
8. 【建】单个源文件 >400 行时考虑拆分；>800 行必须拆分。(源:vflow新增，参考业界实践)

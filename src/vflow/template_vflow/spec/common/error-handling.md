# 错误处理原则

> 适用：所有语言。C++ 异常机制细则见 lang/cpp.md 第 8 节

1. 【规】接口函数参数合法性检查的责任方必须在项目内统一明确；默认由**函数自身**负责。避免双方都不查（漏洞）或双方都查（冗余）。(源:规则81)
2. 【规】对所调用函数的错误返回码要仔细、全面地处理，不得忽略。(源:规则94)
3. 【规】调试打印信息格式项目内统一，至少包含：模块名（或源文件名）、行号。(源:28章)
4. 【规】调试信息支持分级控制：默认只开 fatal/error 级别；debug/info 通过开关控制。(源:28章)
5. 【建】错误信息要能定位问题：报错时带上关键上下文（输入参数值、状态），而非裸的"失败"。(源:vflow新增)

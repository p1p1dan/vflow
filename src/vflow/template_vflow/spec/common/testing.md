# 测试要求

> 适用：所有语言（源:vflow新增，领导硬性要求的落点；原规范无测试章节）

1. 【规】**新增类/公共接口必须配单元测试用例**，至少覆盖正常路径 + 边界条件。纯注释/文档/格式改动豁免。
2. 【规】项目无测试目录时，开发任务中必须先建立测试骨架：`tests/` 目录（镜像源码结构）+ 框架接入 + 构建集成。
3. 【规】框架选型：C++ 非 Qt 项目用 gtest（或 doctest 单头文件轻量方案）；Qt 项目可用 Qt Test（自带零依赖）；Python 用 pytest；C# 用 xUnit。同一项目内统一，不混用。
4. 【规】测试命名：测试文件 `test_<被测模块>.cpp` / `<被测模块>_test.cpp`（项目内统一取其一）；用例名表达"被测行为+条件+预期"，如 `RoundnessCalc_EmptyContour_ReturnsError`。
5. 【规】测试必须独立可重复：不依赖执行顺序、不依赖外部环境（路径/设备/网络）；需要图像样本的测试将样本纳入 `tests/data/` 并随仓库管理。
6. 【规】修 bug 必须先写复现测试（红），修复后转绿——防止同类回归。
7. 【规】验证证据：任务的 verify.md 必须粘贴测试真实运行输出（用例数/通过数），禁止口头宣布"测试通过"。
8. 【建】不设覆盖率硬指标——测试为防错而写，不为凑数而写；优先覆盖核心算法逻辑和边界，而非 getter/setter。
